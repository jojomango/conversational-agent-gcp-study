======

3種agnet簡介

flow and pages: Deterministic
偏向傳統樹狀圖規劃 適合用於嚴謹的流程
如掛失信用卡, 需收集必要用戶資訊, 

playbook: Generative
開發速度快, 設定ai的身份, 遵循什麼步驟, 支援data store實現RAG
適合知識庫問答(FAQ) 意圖引導 與諮詢 (CX Agent Studio)

Vertex AI Agent Builder(CX agent studio)
下一個世代的對話式 AI 開發平台。不再依賴傳統 Dialogflow CX 的 Intents 和 Flows，而是完全建構在現代大型語言模型 (LLMs、Vertex AI) 和 RAG (Data Store) 技術之上。用途專注於讓企業打造高階、純生成式的知識庫對話助理 (Gen AI Agent)，並提供串接各種外部工具 (Tools/Extensions) 的能力。

收費與迭代
[收費方式差異]
- Dialogflow CX (Flow & Playbook)：計費主體主要是「Request/請求次數」（文字）或「語音分鐘數」（語音）。Google 通常會提供新客戶 $600 USD 的試用補助金，方便團隊無痛體驗與整合。
- Vertex AI Agent Builder (CX Agent Studio)：無專屬試用補助。它的計費維度主要是看「Session (工作階段) / Event (對話事件)」，並且還必須疊加上底層消耗的資源，包括 Vertex AI (LLMs) 的 Token 數、Data Store (搜尋) 的查詢與索引費用等。整體營運成本與計費維度皆與 Dialogflow CX 不同。

[迭代方式差異]
- Dialogflow CX Flow：需持續擴充 Training Phrases 與調整樹狀節點。
- CX Agent Studio：重點在於優化 Prompt (Instructions) 與 RAG 資料清洗。

dialogflow 基本上是low-code的圖像化平台, 比較無法像coding的專案讓copilot agent理解全貌
還是需要依賴開發文件來協助維護者理解內容
